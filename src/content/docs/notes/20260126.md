# 📚 CCNA学習メモ：2026/01/26

**学習範囲：P002 ～ P008 / P086**

| ページ | 用語 | 詳細解説 |
| --- | --- | --- |
| **P002** | **ノード** | ネットワーク構成要素（PC、ルータなど）。 |
| **P002** | **リンク** | ノードとノードを結ぶ線。 |
| **P002** | **フロー** | リンク上のデータの流れ。 |
| **P002** | **トラフィック** | ネットワーク上の流れる情報や量。 |
| **P003** | **ＬＡＮ** | 家庭、企業、工場など施設内のネットワーク。 |
| **P003** | **ＷＡＮ** | 本社と支社といった地理的に離れたLAN同士を接続するネットワーク。 |
| **P004** | **インターネット** | 世界中の多くのネットワークが相互に接続された地球規模のネットワーク。家庭や企業のLANなどからインターネットサービスプロバイダ（ISP）を介して接続します。 |
| **P004** | **ＩＳＰ** | インターネットサービスプロバイダ。LANとインターネットを接続する。 |
| **P004** | **イントラネット** | インターネットを利用した企業や組織の内部に構築されたLAN。 |
| **P004** | **ネットワークトポロジ** | トポロジと呼ぶこともある。端末やネットワーク機器の接続形態のこと。 |
| **P004** | **バス型** | 基幹となるケーブル（バス）に各ノードを接続する接続形態。バスに障害が発生するとノード間の通信ができなくなる。 |
| **P005** | **リング型** | 隣接するノードを円状に接続する接続状態。1か所で障害が発生すると、全体に影響が及ぶことがある。 |
| **P005** | **スター型** | スイッチやハブなどの集線装置に各ノードを接続する接続形態。現在のLANネットワークにおける主流の構成。 |
| **P005** | **メッシュ型** | ノード間をそれぞれ接続する接続形態。WANの構成で用いられることが多い、障害に強い構成。 |
| **P005** | **フルメッシュ** | メッシュ型で全てのノードが互いに接続している形態。 |
| **P005** | **部分メッシュ** | パーシャルメッシュともいう。部分的にメッシュとなっている形態。 |
| **P005** | **ハイブリッド型** | ネットワークトポロジを複数組み合わせた接続形態。 |
| **P006** | **通信の方式** | 宛先の数や指定方法により、ユニキャスト、ブロードキャスト、マルチキャストの３つの方式がある。 |
| **P006** | **ユニキャスト** | 特定の宛先にのみデータ送信する方式。１対１での通信。（相手を指定するアドレスを使用） |
| **P006** | **ブロードキャスト** | 不特定多数に対し同じデータを送信する方式。１対多数での通信。（IPアドレスが同じネットワーク内に属していること）途中の集線装置などがデータを複製して転送します。ブロードキャストが多くなると、ネットワークに負荷がかかります。 |
| **P007** | **マルチキャスト** | ある特定のグループに対して同じデータを送信する方式。１対グループでの通信。（グループを指定するアドレスを使用）途中の集線装置などが複製して転送する。通常の無線装置の場合、マルチキャスト宛は全端末に届いてしまうが、端末側でアドレスをみて無視することができるため、ブロードキャストのように負荷はかからない。設定次第で必要としている端末だけに転送するといったこともできるため、効率の良い通信方式。マルチキャストを使用することで、送信元は１つのパケットを送信するだけで必要としている複数の端末にデータを送信でき、通信の効率が向上します。 |
| **P008** | **プロトコル** | データをどのような形式でどのような手順で送受信するかを定めた規則。 |
| **P008** | **通信プロトコル** | ネットワーク上での通信の手順や規約を定めたもの。 |
| **P008** | **プロトコルスタック** | プロトコルシートともいう。複数のプロトコルを階層的に構成したもの。代表例として、OSI参照モデルやTCP/IPモデルがある。 |
